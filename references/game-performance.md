# 游戏帧率参考

当用户在装机、升级或配置检查中问“能跑多少帧”“能不能跑满 240/360/500Hz”“1K/2K/4K 帧数”“打三角洲/打瓦/吃鸡/CS2 帧率”时使用本文件。

## 使用方式

1. 先按正常装机流程确定 CPU、显卡、内存和分辨率。
2. 运行 `scripts/query_game_fps.py` 查离线帧率样本。
3. 只引用脚本返回的已收录样本；查不到时写“离线帧率表暂未收录这个游戏/硬件组合的已核验来源样本”，不要按硬件档位自行推算。
4. `source_default_unknown` 表示公开来源只给出默认/未知画质口径；这是内部数据标签，用户报告里不要直接写出来，改写成“游戏 / 分辨率 大约帧率”。

常用命令示例：

```bash
python scripts/query_game_fps.py --game 三角洲 --resolution 1080p --cpu "Ryzen 7 9800X3D" --gpu "RTX 5070" --memory "DDR5 6000 C30 32GB" --target-fps 300
python scripts/query_game_fps.py --game 3A --resolution 1440p --cpu "Ryzen 7 9800X3D" --gpu "RTX 5070 Ti"
```

## 输出口径

- 帧率只写 1-3 句，放在配置总价和兼容性之后，不要压过装机报告主体。
- 用“参考帧率”“大致区间”“不保证长期稳定”表达，不写成确定承诺。
- FPS 游戏优先写平均帧和 1% low；如果样本只有 FPS 区间，就写“FPS 区间”，不要把区间下限说成 1% low。
- 用户目标是 500Hz/540Hz 时，只有 1% low 样本能支撑“低帧稳住”的说法；公开预测区间只能说明“区间下限也接近/超过目标，但仍需同画质实测复核”。
- 3A 未指定具体游戏时，只用黑神话、赛博朋克等代表游戏做参考；冷门游戏或未收录硬件组合不要编 FPS。
- 帧生成/DLSS/FSR 要单独说明：观感帧率不等于原生渲染帧率，竞技 FPS 不把帧生成当作降低延迟的方案。

## 推荐话术

已收录样本格式：

> 帧率参考：三角洲行动 / 1080p 大约帧率：平均约 X FPS，FPS 区间约 X-Y FPS。这个口径来自公开预测样本，不是实测评测，也不是 1% low；如果你要确认高刷稳定性，下单前还要查同 CPU+显卡+同画质实测。

已收录实测样本格式：

> 帧率参考：按 1080p 竞技低画质，已收录实测样本显示这类组合大致是平均 X-Y FPS，1% low 约 X-Y FPS。这个游戏大场景、烟雾和人数变化会拉低低帧，所以不要只按平均帧承诺高刷目标。

未收录样本：

> 这个游戏/硬件组合离线帧率表暂未收录已核验来源样本，我不编具体 FPS。按硬件方向看，2K 3A 更看显卡，1080p 电竞高刷更看 CPU、缓存和内存延迟；如果你要精确帧数，需要下单前再查同 CPU+显卡的实测。
