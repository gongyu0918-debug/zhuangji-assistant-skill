# 兼容性检查

兼容性由 `scripts/check_compatibility.py` 执行，最终配置必须使用 `--strict`。不要手动替代脚本判断。

脚本覆盖 CPU/主板 socket、内存代际/数量/容量、显卡限长、M.2/SATA 数量、电源功率、显卡供电、散热限高/冷排位、机箱主板版型和电源规格；若机箱或电源有 `psu_length_mm` / `psu_max_length_mm` / `psu_clearance_mm` / `length_mm` 字段，还会复核电源限长。部分 ITX、卧式、小型机箱会另有电源限高、转接支架和硬盘笼空间限制；库内没有结构化字段时必须列为下单前人工复核项，不写成完整通过。

机箱风扇位属于预算和风道复核项，不是硬兼容项，也不作为 `--strict` 阻塞条件。海景房或好看需求使用水冷时，优先读取机箱 `fan_mounts` / `fan_slots_count`：360 水冷占 3 个 120 风扇位，240/280 水冷占 2 个，420 水冷占 3 个 140 风扇位；剩余风扇位按预算预留。用户明确要求水冷夹汉堡/双面风扇时，冷排自带风扇不抵扣加装风扇预算，但必须提示冷排厚度和空间复核。字段缺失时提示用户自行核实机箱风扇信息，并建议预留加装风扇预算。

如果输出包含 `msg` 或 `skipped`，说明数据字段不足；回复用户时必须保留这些复核点，不能写成“完整 11 项全部通过”。

只有 `--strict` 输出 `skipped=0` 且无 `msg/warn/error` 时，才可写“完整通过”。有 `msg`、`warn` 或 `skipped` 时，结论写为“脚本未发现硬不兼容，但仍有下单前复核项”。

电源功率检查只覆盖硬下限。RTX 5080 及以下 850W 高品质 ATX 3.x、RTX 5090/静音/升级余量 1000W+ 等属于选型软规则，仍按 `routing.md` 和最终用途说明取舍。

普通 ATX/MATX 机箱默认选择 ATX 电源。SFX/SFX-L/FLEX/TFX 小电源用于 ITX、小型、NAS、卧式或用户明确复用小电源的场景；装入普通 ATX 机箱即使可用转接支架，也默认按不通过处理，不写成推荐配置。

背插/BTF/Project Zero 不由脚本自动推断。用户明确要求背插时，必须人工确认背插主板与背插兼容机箱，普通机箱不能按背插兼容处理。

主板 M.2 / SATA 数量缺失属于普通装机非阻塞项；单 SSD 整机可列为下单前复核点。用户明确多硬盘、旧 SATA 盘复用、剪辑素材盘、NAS/存储或工作站扩展时，必须补查后再判断。

脚本只检查 M.2/SATA 数量，不自动判断 PCIe 通道共享。多 M.2、PCIe 5.0 SSD、USB4/雷电、采集卡、万兆网卡、声卡或扩展卡场景，必须人工查主板说明书/规格页的 lane sharing：有些插槽会让显卡降到 x8、让 M.2 降到 x2/x4，或禁用 SATA/PCIe 插槽。普通单 SSD 游戏机只列为非阻塞提示，不为此默认升级主板。

显卡的物理 x16 插槽长度和电气 x8 不等于同一概念。RTX5060/RTX5060Ti 等中端卡若官方/规格页为 PCIe x8，在 PCIe 4.0/5.0 平台上不默认视为搭配错误；老 PCIe 3.0 平台、旧主板升级或用户明确追求满血带宽时，再列为复核项。
