# 工作模式

用于处理非标准整机装机场景：旧机升级、配置补全、搭配检查、均衡性评价和硬件原理解释。先判断模式，再决定是否查库和跑兼容性。

## 模式识别

| 用户说法 | 模式 | 处理方式 |
|----------|------|----------|
| 目前配置...想升级 / 还能怎么提升 | 升级模式 | 保留可复用配件，按瓶颈和预算只换必要部分 |
| 给你几项配置，帮我补齐 | 补全模式 | 固定用户已给配件，补齐缺失品类并做兼容性检查 |
| 这套合理吗 / 有没有搭配错误 | 检查模式 | 先判兼容，再判均衡和预算分配 |
| 为什么这样搭 / 为什么不选某件 | 参考模式 | 解释原则，不强行输出整机配置 |
| 为本地 AI 编程/终端 agent 工作流配电脑 / ComfyUI / SD / 文生视频 / PS / 剪辑 / AE / Blender / UE / CAD | 工作流模式 | 先按软件和数据规模判断瓶颈，不套游戏模板 |

用户给出的具体型号优先保留。只有明显不兼容、预算严重失衡、供电/散热/尺寸风险或低端件拖累高端件时，才建议替换。

生产力和本地工作流的通用保护：内存、SSD、电源、散热和机箱风道属于长负载稳定性底线。中高预算配置不只追求贴近预算或跑分，不能为了堆 CPU/GPU 或卡总价而牺牲内存容量、TLC SSD、缓存/项目盘、供电余量和散热风道。若必须取舍，输出“性能优先”和“稳定/扩展优先”两档，并说明差价。

配置摘要或关键配置列表必须写出内存容量、频率和时序；只写“32GB/64GB 内存”不够。64GB DDR5 优先 32GB×2，48GB 优先 24GB×2，96GB 优先 48GB×2；除非用户明确要求插满或没有合理替代，不默认用 4 条 DDR5 套装。硬盘摘要必须写出容量、接口和 TLC/缓存或定位信息。中高端显卡和机箱不能只看脚本刚好通过，显卡限长建议留 20-30mm 以上余量。显卡限长、显卡供电、散热或机箱字段被脚本跳过时，不要写成完整兼容全通过。

## 升级模式

先整理现有配置：CPU、主板、内存、硬盘、显卡、电源、机箱、散热、显示器分辨率/刷新率。信息缺失时不要编，直接列为待确认；但可以先给不依赖缺失项的升级方向。

预算够时优先级：

1. **显卡**: 游戏、3A、2K/4K、AI 绘图等优先升级显卡。升级前必须核对电源功率、显卡供电、机箱限长和散热风道。
2. **硬盘**: 仅当系统盘还是机械硬盘、没有 SSD、SSD 容量太小或老 SATA 明显拖慢体验时优先升级到主流 TLC SSD。已有可用 NVMe SSD 时，不把硬盘排在显卡前。
3. **内存**: 低于 16GB 先补到 16GB；游戏和新平台默认 32GB；多开/生产力再考虑 64GB。低预算或旧机临时过渡可保留 8GB/12GB 候选，但必须说明这是压价取舍。先匹配 DDR 代际和通道，再看频率/时序。
4. **CPU**: 只在当前平台可直接升级、CPU 明显瓶颈、或主玩 FPS/模拟/生产力时优先。若升级 CPU 需要连带换主板和内存，按平台重建方案处理，不默认优先于显卡。
5. **电源、散热、机箱**: 作为显卡/CPU 升级的前置条件。功率、接口、限长、限高不满足时必须先补齐。

升级建议输出：

- 先写当前瓶颈和最值得换的 1-3 项。
- 表格列出建议采购项、参考单价、为什么换、对原配置的要求。
- 原有配件不要重复计入升级预算。需要展示完整清单时，原有/沿用项单价写 `¥0（原有配件，不计入本次预算）`；本次只统计新增采购小计。
- 单独列增量预算、风险点和下单前需确认的型号/尺寸/接口。
- 老平台显卡升级时，不默认推荐 RTX5050；RTX5060 只贵几百且预算够时优先上 RTX5060。若原机没有 SSD、只有机械盘或老 SATA SSD，先把主流 TLC NVMe SSD 写入升级项。
- 升级模式里的 SSD 也遵守主流 TLC 和候选池权重规则；查询时用 `--min-capacity 1000|2000 --sort tier`，再按预算选择。预算不足时给“显卡优先”和“均衡可靠”两档，不要用规格信息不完整或系统盘场景不匹配的 SSD 凑满预算。

## 补全模式

用户给了部分配件时，不要擅自替换已给项目。按缺失品类补齐：散热、电源、机箱和硬盘也必须补；海景房要预留风扇预算。

处理步骤：

1. 把用户已给配件列成“沿用/固定项”，单价写 `¥0（已定/已拥有，不计入本次预算）`；另列新增采购小计。只有用户明确要估算整机价值时，才单独给整机参考总价。
2. 用已知 CPU/主板/内存代际收窄候选；不要混 DDR4/DDR5，不要 AM5 配 DDR4。
3. 补齐缺失件后运行 `scripts/check_compatibility.py --strict`。如果用户给的是库里匹配不到的型号，说明无法脚本完整校验，并给人工核对项。
4. 总价区分“新增采购小计”和“整机参考总价”。已给配件无价格时，不要把总价写成完整整机价。

## 检查和参考模式

检查配置时按两层判断：

1. **硬兼容**: socket、DDR 代际、M.2/SATA、显卡限长、散热限高、冷排位、电源功率、显卡供电、主板版型、电源规格和背插/ITX形态。能匹配离线库时跑 strict；不能匹配时列人工核对点。
2. **软均衡**: CPU/GPU 档位、主板供电、电源余量、内存容量/频率、SSD 颗粒和容量、预算是否花在用户真正关心的地方。

明显不均衡的例子：

- i3-12100/12400 搭 RTX 5080/5090：4K 也不是绝对不能亮机，但高刷、FPS、1% low 和后续显卡利用率会被 CPU/平台拖累；除非只是临时过渡，否则应提高 CPU/平台或下调显卡。
- i5-14600K/KF、i7 或更高搭 H610、无 VRM 散热片或供电规格偏弱的入门 B760：容易被功耗墙、VRM 温度或 BIOS 默认限制压住性能。优先选供电和散热更完整的 B760；Z790 主要用于超频、更多 I/O 或用户明确要求，不是 14600K + RTX5070 级补全的默认项。
- 高端显卡搭低瓦数旧电源、无原生 12V-2x6/12VHPWR 或机箱限长不足：即使核心配件档位正确，也必须先解决供电和尺寸。

回答“为什么”时用结论先行：

- **AMD DDR5 6000 甜点位**: AM5 日常优先 DDR5-6000 CL30/32，因为低延迟和稳定性平衡好；更高频率可能进入分频或需要更强 IMC/主板调校，游戏收益不稳定。X3D 因大缓存对内存更不敏感，不应为极高频内存牺牲显卡/CPU预算。
- **9800X3D 配 B850/X870**: 不是硬性必须。AMD 官方支持 A620/B650/B850/X870 等 AM5 芯片组；高预算优先 B850/X870 是为了 Ryzen 9000 开箱 BIOS、供电散热、PCIe 5.0、USB/Wi-Fi 和扩展性更稳。预算紧时，供电足、BIOS 新的 B650/B650E 也可以。
- **TLC 优先于 QLC**: QLC 主要优势是容量成本，常见短板是持续写入和耐久余量；普通装机系统盘优先 TLC，QLC 只在大容量冷数据盘或预算非常紧时说明取舍后使用。
- **高预算 SSD 选择**: 9000 元以上整机和高端补全默认用三星、致态、宏碁掠夺者、雷克沙、西数、铠侠、英睿达、海力士等热门采用/高规格 TLC 候选。预算备选或价格异常时可换用常见装机或新兴特色候选，但必须说明容量、颗粒、保修、渠道数据或价格取舍。
- **电源 16pin 字段缺失**: 离线库未标记原生 16pin/12V-2x6 时，不能直接写成搭配错误。若型号/规格显示 ATX 3.x、PCIe 5.x 或可用合规转接线，按“下单前复核线材”处理；只有外观理线、白色海景房或用户明确不要转接时，才把原生线材作为优先项。
- **电源规格和限长**: 普通 ATX/MATX 机箱默认用 ATX 电源；SFX/SFX-L/FLEX/TFX 小电源装普通 ATX 机箱通常需要转接支架且线材长度可能不理想，默认不通过/不推荐。ITX、小型、卧式、NAS 或用户明确复用小电源时，再按机箱电源规格和限长/限高判断，并保留下单前复核项。
- **超长显卡和机箱**: 显卡长度接近 380-400mm 时按超长卡处理。机箱限长等于显卡长度即使脚本 PASS，也要提示线材、前置风扇/冷排和安装余量风险；默认优先限长 420-430mm 以上或风道更宽松的机箱。中小机箱优先 B850M/B760M/B860M 等 M-ATX 主板。
- **异常低价和展示型机箱字段冲突**: 机箱、电源、散热价格明显低于同类时只作备选并提示复核，不用来硬贴预算。用户明确不要海景房或要低调无光时，型号/ID 写有海景房、全景、无立柱、鱼缸、曲面玻璃、副屏、LCD、O11、TK 等展示型关键词的候选应排除，即使 `is_showcase` 字段为 false。
- **报告话术**: 面向用户不要写 `check_compatibility.py --strict 返回 overall=pass`、`channel_quote`、`needs_market_quote` 这类内部表达。改写成“兼容性检查完成，未发现硬不兼容”“价格为离线渠道参考价，需下单前复核实时价格和库存”。如果有跳过项，就写“仍需人工复核显卡限长/电源线材/机箱风扇位”等具体事项。
- **背插/BTF/Project Zero**: 背插主板把主要接口放到背面，主要价值是理线和展示效果，不是性能提升。必须搭配明确支持背插主板开孔和背部走线空间的机箱；涉及华硕 BTF/GC-HPWR 或同类隐藏显卡供电时，还要确认显卡/主板/电源线材生态是否匹配。用户未明确要求背插时，不为外观自动切到背插。
- **ITX 主机**: ITX 不是单纯把主板换小。必须同时核对 Mini-ITX 主板、机箱电源规格、显卡长宽厚、CPU 散热高度/冷排位、风扇位、硬盘位和线材空间。预算相同下 ITX 通常比普通 M-ATX/ATX 更贵、更热或更难装，输出时说明性能、噪音、温度和价格取舍。

## 用途特例

### 直播、推流、录制

用户提到直播、推流、OBS、录制、剪辑直播素材时，优先考虑 Intel 非 F 型号或 Core Ultra 带核显型号。核显可提供 Quick Sync 编解码、故障排查备用输出和转码余量；如果同时使用 NVIDIA 显卡，可说明 NVENC 也是可行方案。只有用户明确只要最低价时，才把 F/KF 作为压预算取舍。9000 元左右的直播游戏主机优先从 i5-14600K/14600 等级开始试配，而不是只按低价 i5 压预算。

输出时说明：

- 直播推流优先 Intel 带核显 + NVIDIA 显卡，兼顾 Quick Sync 和 NVENC。
- 若用户主玩游戏且预算有限，可在 Intel 带核显和 AMD X3D 之间说明取舍：Intel 偏推流/生产力便利，X3D 偏高帧游戏。

### 热门 FPS / 电竞网游

这些说法都按 FPS 高帧路由识别：三角洲行动/三角洲、绝地求生/吃鸡/PUBG、CS2/CS GO/GO、无畏契约/瓦/打瓦/Valorant。P 社和大型模拟策略游戏（群星、维多利亚、欧陆风云等）也按 CPU 缓存/后期模拟帧优先处理。

规则：

- 预算允许时优先 X3D 处理器，尤其 9800X3D；能上 9800X3D 时不默认退到 7800X3D。更高预算且兼顾生产力时再考虑 9950X3D。
- 显卡按分辨率和刷新率“够用并留余量”，不要为了旗舰显卡牺牲 CPU、内存延迟和散热稳定性。2K FPS/吃鸡/CS2 这类场景可用 RTX5070/5070Ti 配平时，不强行堆到 RTX5080。
- 内存优先低延迟双通道：AM5 默认 DDR5-6000 CL30/32，Intel 可看 DDR5 6400-7200 或成熟 DDR4 3600/3200 取性价比。
- 输出要关注 1% low、CPU 限制、温度稳定性和显示器刷新率，不只看平均帧。

### 本地 AI 编程/终端 agent 硬件配置

用户明确是在为本地 AI 编程/终端 agent 工作流配电脑、升级电脑或评估硬件需求时，才进入本节。典型说法包括“给 Codex/Claude Code/opencode/OpenClaw/Hermes 配电脑”“多 agent 并发主机”“本地项目索引/离线知识库/大量工具链并发吃配置吗”。如果用户只是问如何安装、运行或使用某个 agent、插件、命令或工作流，不触发装机配置。

规则：

- 优先级为内存、SSD、CPU，其次才是显卡；但只要用户场景和 AI 辅助、embedding、向量库、本地小模型、ComfyUI 或视频生成有关，预算允许时默认保留一张 NVIDIA 显卡作为 CUDA/本地模型弹性，不把 Intel Arc 作为 AI 相关默认显卡。用户明确纯云端、纯编译或极限省钱时，才降低显卡权重。
- 官方最低运行门槛不等于装机推荐。Claude Code 官方最低可到 4GB+ RAM；OpenCode/Codex 这类终端 agent 主要依赖网络模型和本地项目工具链。实际装机按“IDE + 浏览器 + 终端 agent + 索引/缓存 + Docker/测试”的并发负载估算。
- 默认 32GB 起步；多项目、多容器、浏览器和 IDE 常驻建议 64GB。多 agent 并发、本地向量库/知识库、多个沙箱或后台服务常驻，可建议 96GB/128GB；查询内存时用 `--min-capacity 64|96|128 --sort tier` 收窄，再看频率、时序、套条数量和候选池权重。不要为纯 agent 盲目牺牲 SSD 和 CPU。
- SSD 优先 2TB 主流 TLC NVMe；大量索引、缓存、日志、向量库、模型或多仓库时建议第二块 SSD 或 4TB，查询时用 `--min-capacity 2000|4000 --sort tier`，并优先看 TLC、耐久和持续写入，不要用 QLC 系统盘。
- CPU 优先高单核和足够核心数，Intel 带核显对排障/编码有价值；AMD 也可用，按预算和平台扩展性取舍。
- 如果用户实际想跑本地 LLM、embedding 或重排序模型，再单独进入本地模型路线，显存/内存需求会明显上升。

### ComfyUI / Stable Diffusion / 文生视频

用户说 ComfyUI、SD、Stable Diffusion、文生图、LoRA、本地绘图、文生视频、AnimateDiff、Flux、SVD 等，必须按本地生成式 AI 路线处理。

规则：

- 必须优先 NVIDIA 显卡和显存；同预算先保显存和 CUDA 生态，再看 CPU 核心数。AMD/Intel 显卡只有用户明确要求或预算极低时作为取舍，并说明生态风险。
- 文生图入门至少 12GB 显存更稳；文生视频、较大模型、多 ControlNet/高分辨率优先 16GB/24GB+ 显存。
- RTX5070Ti 和 RTX5080 都是 16GB 显存时，不要只因型号更高自动上 RTX5080；需要比较价格、显存带宽、功耗、散热、SSD/内存/主板是否被挤压。文生视频或重度生成仍优先显存容量和带宽，预算不够时可把 RTX5070Ti + 更大 SSD/更好电源作为均衡方案。
- 内存默认 32GB 起步，文生视频、多模型常驻或生产力混用优先 64GB；查询内存时用 `--min-capacity 64 --sort tier` 避免被 32GB 低价候选带偏。
- SSD 默认 2TB 主流 TLC NVMe；模型库、素材、缓存较多时优先 4TB 或双 SSD，查询时用 `--min-capacity 2000|4000 --sort tier`，且避免 QLC 系统盘。
- CPU 够用即可，但不要用太低端 CPU 拖慢预处理、解码、批处理和多任务。

### CAD / 建模绘图

用户只说 CAD 时，先问或自行列出假设：AutoCAD 2D、AutoCAD 3D、SolidWorks/Inventor/Revit、装配规模、是否专业商用和是否需要认证显卡。

规则：

- AutoCAD 2D/轻 3D 优先高频 CPU、32GB 内存和主流 TLC SSD；显卡不用盲目堆高，基础独显或中端 NVIDIA 足够。
- AutoCAD 大型 3D、点云、多屏和复杂模型提高显卡和内存优先级，但仍保高单核 CPU。
- SolidWorks/复杂装配优先高频 CPU、32GB/64GB 内存、SSD；装配越大内存越优先。
- 专业商用、需要官方支持、RealView/Ambient Occlusion 或复杂装配视口时，说明 RTX PRO/工作站卡的价值；预算有限或兼顾游戏时可用 GeForce，但要明示认证和支持取舍。
- CAD 工作站不建议超频式不稳定方案；电源、散热、主板供电要保守。

### PS / Lightroom / 图片后期

用户说 PS、Photoshop、Lightroom、修图、摄影后期、批量出片时，按图片后期硬件路线处理。

规则：

- 高频 CPU 优先，核心数不需要无限堆；预算足够优先较新的 Ryzen 7/9、Intel i5/i7/Core Ultra 中高频型号。
- 内存 32GB 起步；大尺寸 RAW、全景、堆栈、多图层 PSD 建议 64GB。
- SSD 优先主流 TLC NVMe，系统/软件和素材可一块 2TB 起步；素材量大时加第二块 SSD 或大容量仓库盘。
- 显卡需要支持并稳定，但纯修图不为旗舰显卡牺牲 CPU/内存/SSD；中端 NVIDIA/AMD 独显通常足够，多 4K 显示器或 AI 滤镜再提高显存。

### PR / 达芬奇 / 视频剪辑调色

用户说 PR、Premiere、视频剪辑、B站剪辑、4K 剪辑、达芬奇、DaVinci、调色时，先问或假设素材分辨率、编码、位深、特效、代理和是否多软件并开。

规则：

- Premiere/H.264/H.265 素材优先 Intel 带核显 + NVIDIA 显卡，兼顾 Quick Sync 和 NVENC；RAW/多特效/多软件时提高 CPU、内存和显卡档位。
- Core Ultra 200S 平台优先 Core Ultra 5 250K/250KF Plus、Core Ultra 7 270K 或库内同档 Core Ultra 7；Core Ultra 9 只在重度多任务、复杂工程或预算明确足够时使用。
- DaVinci Resolve 更偏 GPU/显存，调色、降噪、Fusion、AI 功能优先 NVIDIA 和显存；4K 建议 12GB+ 显存，6K/8K 或重降噪建议 16GB/20GB+。
- 内存按分辨率给足：1080p 可 32GB 起步，4K 剪辑建议 64GB，6K/8K 或 AE/PS/PR/达芬奇并开建议 96GB/128GB+。
- SSD 不只看容量：至少 2TB 主流 TLC NVMe；预算足够时分系统/软件、素材项目、缓存/暂存盘。高码率素材和缓存优先高速 TLC，不用 QLC 系统盘。
- 主板也是剪辑平台重点：素材导出、外置高速盘、采集卡、多硬盘和缓存盘会吃 I/O。AMD 优先 X870/X870E 并复核 USB4、M.2、SATA；Intel 优先 Z790/Z890 或同等级高扩展主板，并复核高速 USB/雷电扩展、M.2、SATA 和 PCIe 插槽。
- 电源、散热和机箱风道按长时间渲染负载留余量，不用刚好够的电源和入门散热压高端 CPU/GPU。

### AE / 动效合成

用户说 AE、After Effects、动效、合成、MG 动画时，按 CPU、内存和缓存盘优先处理。

规则：

- CPU 对整体体验很关键，优先高性能新平台；大量渲染和多软件并开再提高核心数。
- 内存 64GB 作为主流建议；4K 以上、复杂合成、多软件并开或长时间预览建议 128GB。
- 高速 TLC NVMe 缓存盘很重要；预算允许时把缓存/项目盘与系统盘分开。
- 2D 动效不盲目堆显卡；3D、Advanced 3D、粒子、GPU 插件较多时优先 NVIDIA 和更大显存。

### Blender / 3D / 渲染

用户说 Blender、C4D、Maya、3D 建模、动画、渲染时，先按建模动画、GPU 渲染、CPU 渲染、流体/布料仿真分流。

规则：

- 建模和动画看高频 CPU、稳定视口和足量内存；不需要为建模本身堆极高核心数。
- GPU 渲染优先 NVIDIA、CUDA/OptiX 和显存；项目复杂度不明时宁可提高显存，不要只看核心性能。
- CPU 渲染、流体仿真、烘焙等吃多核心时，提高核心数和散热，预算高再考虑 HEDT/工作站平台。
- 内存小项目 32GB 起步，大场景/贴图/仿真/多软件建议 64GB/128GB。
- SSD 至少 2TB 主流 TLC NVMe；素材库、贴图库、缓存和工程多时加第二块 SSD 或大容量仓库盘。

### UE / Unity / 游戏开发

用户说 UE、Unreal、Unity、游戏开发、虚幻、Shader 编译、打包时，按引擎开发工作站处理。

规则：

- CPU 是编译代码、Shader、烘焙和导入资源的关键；项目越大越需要更多核心，但普通用户不默认上 HEDT。
- AMD 非 X3D 多核心路线预算足够时优先 Ryzen 9 9950X；如果预算不足或更看重单 CCD 稳定表现，可用 Ryzen 7 9700X。Ryzen 9 9900X 只有在价格优势或核心数收益明确时作为取舍。
- GPU 决定编辑器视口、实时预览、光追和目标画质；UE5/大型场景优先 NVIDIA 和 12GB/16GB+ 显存。
- 内存 32GB 为入门，较大项目、烘焙、多工具并开建议 64GB+。
- SSD 默认 2TB 主流 TLC NVMe；大型工程、Derived Data Cache、素材库和构建输出多时建议第二块 2TB/4TB SSD。
- 若用户同时玩 3A，先满足开发项目规模，再按游戏分辨率补显卡档位。

## 高预算配平

高预算仍要均衡，不把钱浪费在装饰件或重复堆料上。

| 预算 | 默认方向 | 注意 |
|------|----------|------|
| 15000 | 游戏/FPS 优先 9800X3D + RTX 5070 Ti/5080 档；32GB DDR5-6000 低延迟；2TB 主流 TLC SSD；850W+ ATX 3.x 金牌电源 | 2K FPS 不强行上 5080；如果 5080 超预算，优先保 9800X3D、显卡档位和电源下限，外观件降级 |
| 20000 | 9800X3D + RTX 5080 高规格；2TB/4TB 高端 TLC SSD；32GB/48GB 高接受度内存；850W/1000W ATX 3.x 电源按静音和升级余量取舍 | 纯游戏不默认上 9950X3D；RTX5080 通常 850W 足够，1000W 属富余或升级余量；生产力/直播剪辑再考虑更多核心和 64GB 内存 |
| 25000+ | 4K 高刷/生产力按 RTX 5090/5090D、9950X3D/高端 Intel 分流 | 必须核对机箱风道、电源接口、显卡长度和散热，不用低端板电源硬拖旗舰件 |

预算分配原则：CPU 和显卡优先，CPU 够用但不能拖显卡；主板、电源、散热、机箱是稳定性底线；SSD/内存按容量和可靠性足够即可，不为了跑分挤占显卡预算。纯游戏 32GB DDR5 超过 1500 元必须解释价值；没有用户指定、48GB/64GB 或特挑低时序理由时，必须改选 800-1300 元主流套条，把钱留给显卡、电源和散热。

用户明确 ROG 全家桶、华硕全家桶、白色旗舰或展示向整机时，可以把高价 ROG/TUF/同生态配件作为主方案，但要把溢价说明成人能理解的取舍：外观统一、生态联动、散热规格、屏幕/灯效或品牌系列，而不是性能必需。高价不设硬上限，低价异常仍要复核。预算明显不足以同时保证核心性能、电源、散热和机箱风道时，不硬凑零散旗舰件；先说明预算缺口，再给同生态非旗舰、TUF/PRIME 或性能优先替代。
