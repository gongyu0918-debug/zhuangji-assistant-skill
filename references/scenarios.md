# 场景细则

用于处理具体装机场景。先由 `routing.md` 判断预算、用途和关键约束，再按需读取本文件对应章节；升级、补全和检查模式仍先读 `workflows.md`。

本文件只处理硬件配置。用户只是问软件安装、使用教程、插件、报错、模型下载、系统设置或工作流配置时，不因出现 ComfyUI、PS、PR、AE、Blender、UE、CAD、本地 agent 等关键词而进入装机方案。

## 用途默认路由

- 3A: 低U高显，优先显卡/显存/散热；CPU够用即可。
- 网游: CPU和显卡均衡，优先高频低延迟内存。
- 热门 FPS / 电竞网游: 优先 X3D 处理器、低延迟内存、温度稳定和 1% low；显卡按分辨率/刷新率够用即可，不盲目堆旗舰显卡。预算紧且主玩三角洲、PUBG/吃鸡、CS2/GO、无畏契约/瓦时，可把 AM4 X3D + B550 + DDR4 3600 作为低价例外路线和 Intel D4/AM5 对比；查询 CPU 和主板时显式使用 `--socket AM4 --sort tier`，让高采用率的 5800X3D 优先于其它 AM4 X3D 候选。
- 用户问具体帧率或跑满刷新率时，读取 `references/game-performance.md` 并运行 `query_game_fps.py`。只引用离线帧率表的已收录样本；未收录游戏或硬件组合不要编 FPS。
- 直播推流: 优先 Intel 带核显 CPU + NVIDIA 显卡，兼顾 Quick Sync 和 NVENC；只有用户明确最低价时才考虑 F/KF。
- 办公: 高U低显或核显，看CPU/多屏/内存/SSD/静音。
- 生产力: 按软件分流（剪辑/AI/渲染是否吃GPU）。
- 多开: CPU核心/内存容量/SSD/电源稳定性优先。
- 为本地 AI 编程/终端 agent 工作流配电脑: 内存、SSD、CPU 优先；只要需求涉及 AI 辅助、embedding、向量库、本地小模型、ComfyUI/SD 或 CUDA 弹性，预算允许时默认保留 NVIDIA 显卡，不把 Intel Arc 作为 AI 相关默认显卡。用户只是问如何使用某个 agent、插件或命令时，不进入装机配置路线。
- ComfyUI/SD/文生视频: 必须 NVIDIA 显卡优先，先看显存和 CUDA 生态，再补足 SSD、内存和 CPU。
- PS/Lightroom: 高频 CPU、32GB 起步内存、主流 TLC SSD 优先；中端 NVIDIA/AMD 独显即可，不为纯修图堆旗舰显卡。
- PR/视频剪辑: CPU、硬件编解码、内存、显卡、SSD 和主板 I/O 都重要；H.264/H.265 素材优先 Intel 带核显 + NVIDIA，4K 起步 64GB 内存和 8GB+ 显存；素材导出/外置高速盘/多硬盘工作流提高 X870/X870E 或 Z790/Z890 等高扩展主板权重。
- 达芬奇: 更偏 GPU/显存；4K 调色/降噪优先 12GB+ 显存和 NVIDIA，内存/SSD 随分辨率提高。
- AE: CPU 和内存优先，缓存盘要好；2D 动效不盲目堆显卡，3D/Advanced 3D 再提高 NVIDIA 显卡和显存。
- Blender/3D: 建模动画看高频 CPU 和显卡视口，GPU 渲染优先 NVIDIA/显存，CPU 渲染/流体仿真再提高核心数。
- UE/Unity: 引擎视口看显卡和显存，代码/Shader 编译看 CPU 核心数，项目体量决定 32GB/64GB+ 内存和 2TB+ SSD。
- CAD/建模绘图: 先问软件和工作类型。AutoCAD 2D 优先高频 CPU、32GB 内存和 SSD；3D/大模型/装配提高显卡和内存；SolidWorks 专业环境要说明认证 RTX PRO 和 GeForce 的取舍。

## 预算场景细则

- 4000 元 3A 属于入门新件边界，目标是贴近 4000 元和 1080p 可玩，不要强称 2K/高画质。为贴预算可接受 16GB 内存、512GB SSD、Arc B570/RTX5060/同档显卡；RTX5050 只在预算硬卡时作为压价备选，并明确后续升级 32GB 内存和 1TB SSD 的取舍。
- 6000 元 3A 在可信行情地板较高时不要硬堆到 7000+。若 RTX5060 + 32GB + 1TB + 可靠电源/机箱无法贴近 6000，先输出贴预算方案（可接受 16GB 或 512GB 作为临时取舍、显卡保持 RTX5060/同档优先），再输出超预算均衡方案（32GB + 1TB + 更稳电源），并明确差价和后续升级顺序。
- 8000 元白色海景房优先保显卡档位和整机预算贴近；如果为了白色件/海景房外观回退到 DDR4 或较低 CPU，需要把它写成取舍，不要说成同价位最优平台。白色/黑色海景房为了好看可以优先查水冷，但预算不足时允许回落到同色风冷 + 风扇预算，不能为水冷挤掉显卡档位或电源底线。
- RTX5060Ti 16GB 当前更适合 AI/生产力、显存敏感游戏或用户明确要更长使用周期。纯游戏/主流玩家如果 16GB 版本价格已贴近 RTX5070，不默认把 RTX5060Ti 16GB 当甜点位；预算紧张用 RTX5060Ti 8GB 或 RTX5060/AMD/Intel 同价位显卡压预算，预算可加时优先比较 RTX5070。
- ROG 全家桶、华硕全家桶或旗舰生态需求先核对预算是否能覆盖核心性能、电源、散热和机箱风道。预算明显不足时，不硬凑少数旗舰件；先说明预算缺口，再给同生态非旗舰、TUF/PRIME 或性能优先替代方案。
- 为本地 AI 编程/终端 agent 工作流配电脑、ComfyUI/SD、PS/剪辑/AE、Blender/UE、CAD 等生产力需求不要套游戏配平比例；先按工作流瓶颈分配预算，再补游戏能力。
- 本地大模型/LLM、长上下文推理、多模型并发或明确 48GB/64GB 显存仍不够的场景，预算 13 万元以上可用 `--include-workstation-gpu` 查询 RTX PRO 6000 Blackwell Workstation 96GB；必须同时提高内存(128GB/192GB+)、SSD(4TB+ 高规格 TLC)、电源(1200W+ 高品质 ATX 3.x/PCIe 5.x)和全塔/工作站机箱风道权重。预算不到或只是本地终端 agent/普通 ComfyUI 时，不默认推荐 10 万元级工作站卡。15000 元以上 ComfyUI/SD 若不为显卡/显存硬卡预算，CPU 至少比较 R7 9700X；如果 9700X 只小幅超预算或能通过非核心外观件回收预算，优先给 9700X。R5 9600X 只作为“保显卡/保显存/保 64GB 内存或大 SSD”的压预算取舍，并写出 9700X 替代价差。
- 本地 LLM + ComfyUI 同时出现时，先说明显存是核心瓶颈。预算 30000 左右如果 32GB 显存 RTX5090/5090D 会让整机明显超预算，不要把 RTX5080 16GB 写成理想本地 LLM 方案；应给“预算内 ComfyUI/轻量模型方案”和“32GB 显存超预算方案”两档。

## 显卡特殊情况

- RTX5090、RTX5090D 和 RTX5090D V2 不视为同一候选。RTX5090D V2 是 24GB/384-bit 规格，和 32GB/512-bit 的 RTX5090/RTX5090D 分开报价、排序和解释；纯游戏可按实测价格、散热和售后比较，ComfyUI、文生视频、本地模型、3D 渲染等显存或带宽敏感场景要优先说明 24GB 与 32GB 的取舍。
- RTX5060Ti 8GB 和 RTX5060Ti 16GB 视为不同显存候选，不按同一张卡合并。配置报告必须写显存容量；AI/生产力、2K 高纹理或更长使用周期优先比较 16GB，低预算纯游戏才把 8GB 作为压价取舍。
- 用户明确显卡芯片时，显卡查询加 `--gpu-chip RTX5060Ti|RTX5080|RTX5090D V2`，并写完整 Ti/D/V2/XT/GRE 等后缀，避免 RTX5070 与 RTX5070Ti、RTX5090D 与 RTX5090D V2 混查；明确要 16GB/24GB/32GB 显存时再叠加 `--min-vram 16|24|32`，例如 RTX5060Ti 16GB 用 `--gpu-chip RTX5060Ti --min-vram 16`。
- 高端 RTX5080/RTX5090/RTX5090D/RTX5090D V2 中存在少量水冷显卡，型号通常带水神、水雕、水超龙、水夜神、Neptune、WATERFORCE、LIQUID、LC 等明确词；普通显卡查询默认加 `--gpu-cooling air`，只有用户明确需要水冷显卡、低噪展示效果，或预算足够且接受复杂装配时才查 `--gpu-cooling liquid`。CPU 水冷不等于显卡水冷。选择后必须复核机箱额外冷排位/风扇位、CPU 水冷冲突、冷排厚度、显卡长度和线材空间。
- Arc B570/B580 可以作为低预算、显存容量或 AV1 编码取舍候选；纯 3A 游戏默认仍优先同预算 NVIDIA/AMD 主流游戏生态。采用 Arc 时必须说明驱动、老游戏和个别 3A 兼容性需要复核。

## 色系、海景房、ITX 和背插

- 黑色: 优先黑色/默认黑色。
- 白色: 必须白色型号或明确白色版本。
- 海景房: 展示型机箱(玻璃侧透)，需要查冷排位、显卡限长、白色件价格溢价，并预留风扇预算。
- 白色/黑色海景房、好看、展示向: 散热默认先查 240/360 水冷；预算充足且不牺牲显卡、电源、SSD、内存底线时，可优先带屏/LCD 水冷。预算紧、纯性能或无光需求时，水冷和屏幕只作为可选审美升级。
- 9800X3D、14600K/KF 以上、Core Ultra K/KF、高预算 X3D 或长负载生产力平台，不默认用 120SE/入门单塔风冷压预算；至少比较强双塔、240/360 水冷或同级散热。9000 元以上且不是极限压预算时，散热优先看 250-400 元级强风冷/360 水冷，再用更低价散热作压预算方案。白色/黑色海景房好看需求优先水冷。
- 散热预算按整机占比复核：普通纯性能约 1.5-4%，海景房/白色/好看约 3-7%，ITX/静音/旗舰或带屏水冷约 4-8%。低于区间要说明 CPU 功耗低或采用强风冷；高于区间要说明屏幕、静音、外观或长负载需求。
- 不要RGB: 不花预算在灯效/海景房/白色溢价和装饰风扇上。
- 不要海景房 / 不要玻璃 / 低调无光: 除了看 `is_showcase=false`，还要排除型号或 ID 中包含海景房、全景、无立柱、鱼缸、曲面玻璃、副屏、LCD、O11、TK 等明显展示型关键词的候选；字段和型号冲突时以型号语义为准。
- 海景房 / 好看 / 白色海景房 / 黑色海景房: 查机箱时优先读取 `fan_mounts` 或 `fan_slots_count`。使用 360 水冷时默认占 3 个风扇位，240/280 水冷占 2 个，420 水冷占 3 个；输出中保留“水冷占用位”和“剩余风扇预算预留”。字段缺失不是阻塞条件，不要编数量；提示用户自行核实机箱风扇信息，并建议预留加装风扇预算。需要具体风扇时单独查 `--category fan`，海景房优先同色系 RGB，优先同系列正页/反页搭配；预算充足和展示向可提高积木风扇、带屏风扇权重。
- 中小机箱: 优先 M-ATX 主板，例如 B850M、B760M、B860M；显卡长度、前置风扇/冷排、电源线材走线要留余量，不采用“刚好等于限长”的方案作为默认优先。
- ITX 主机: 只有用户明确要求 ITX、小钢炮、A4、桌面小主机、NAS/HTPC 等紧凑形态时进入。必须同时收窄 Mini-ITX 主板、ITX 机箱、机箱指定电源规格(SFX/SFX-L/FLEX/TFX/少数 ATX)、CPU 散热限高或冷排位、显卡长度/厚度、风扇位和硬盘位；缺任一关键尺寸时不能写完整通过。
- 背插/BTF/Project Zero: 只有用户明确要求背插、少线、无线感、BTF、Project Zero 或海景房极简理线时进入。必须确认背插主板、背插兼容机箱和对应开孔/背部走线空间；华硕 BTF/GC-HPWR、微星 Project Zero 等生态可作为路线，但不要混写成通用兼容。普通主板按普通方案处理，背插不是性能优先项。

## 机箱和风扇

- 机箱是必选配件，必须计入总价。优先选有价格的兼容机箱；无价格机箱只能作为备选并标注总价不完整。
- 机箱预算按整机占比复核：普通风道约 3-6%，海景房/白色/好看约 5-9%，ITX/静音/ROG/旗舰约 6-12%。低于区间不直接禁止，但必须复核板材、风道、限长、风扇、理线和噪音；高于区间要说明外观、风道、背插、ITX 或长期复用价值。
- 用户没有外观、静音、ITX、背插、海景房、全家桶或长期复用需求时，机箱不要主动堆太高预算；常规中高端配置优先把机箱控制在约 1200 元以内，把预算留给 CPU、显卡、内存、SSD、电源和散热。超过时必须说明风道、做工、风扇套装、展示效果或扩展价值。
- 显卡限长建议留 20-30mm 以上余量；长度接近 380-400mm 的显卡按超长卡处理，优先选择限长 420-430mm 以上且风道充足的机箱。脚本刚好 PASS 仍要提示实装、线材和前置风扇/冷排复核。
- 海景房默认需要补风扇。若用户没有指定水冷，按机箱风扇位预留 3-6 把风扇预算；若用了水冷，冷排自带风扇可抵扣对应风扇位。用户明确说“水冷夹汉堡/夹汉堡/双面风扇/push-pull”时，不扣减水冷预留位，按当前色系补颜值风扇，并提示冷排厚度、显卡余量和线材空间需要复核。
- 海景房风扇预算普通约占 1-4%，高端联动风扇或旗舰展示约 2-5%；预算紧张时可以先预留预算并说明后补，不要编库内没有的具体风扇型号。需要 RGB 时注意样式一致性，优先同款或同系列的正页/反页版本，不混搭明显不同灯圈/框架样式。
- 风道机箱、封闭机箱、低调无光或养宠防毛场景，不默认补满风扇位；风道机箱通常自带大风扇且可用风扇位较少。若 `fan_mounts` / `fan_slots_count` 缺失，不推荐具体风扇数量和型号，只提示按机箱实物复核。确实要加风扇时，优先无光、同尺寸、同系列或转速定位接近的风扇，避免单独加入明显高转高速风扇导致噪音突兀。
- 选择水冷时必须复核机箱冷排位、冷排厚度、显卡限长和顶部/侧部空间；带屏水冷还要确认屏幕不会和用户“无光/低调”诉求冲突。
- 离线库风扇数据不足时，不要编具体风扇型号；用"风扇预算预留"写入总价或备注，并建议下单前确认机箱自带风扇数量、风扇位和预装风扇规格。

## 直播、推流、录制

生产力和本地工作流的通用保护：内存、SSD、电源、散热和机箱风道属于长负载稳定性底线。中高预算配置不只追求贴近预算或跑分，不能为了堆 CPU/GPU 或卡总价而牺牲内存容量、TLC SSD、缓存/项目盘、供电余量和散热风道。若必须取舍，输出“性能优先”和“稳定/扩展优先”两档，并说明差价。

用户提到直播、推流、OBS、录制、剪辑直播素材时，优先考虑 Intel 非 F 型号或 Core Ultra 带核显型号。核显可提供 Quick Sync 编解码、故障排查备用输出和转码余量；如果同时使用 NVIDIA 显卡，可说明 NVENC 也是可行方案。只有用户明确只要最低价时，才把 F/KF 作为压预算取舍。9000 元左右的直播游戏主机优先从 i5-14600K/14600 等级开始试配，而不是只按低价 i5 压预算。

- 直播推流优先 Intel 带核显 + NVIDIA 显卡，兼顾 Quick Sync 和 NVENC。
- 若用户主玩游戏且预算有限，可在 Intel 带核显和 AMD X3D 之间说明取舍：Intel 偏推流/生产力便利，X3D 偏高帧游戏。

## 热门 FPS / 电竞网游

这些说法都按 FPS 高帧路由识别：三角洲行动/三角洲、绝地求生/吃鸡/PUBG、CS2/CS GO/GO、无畏契约/瓦/打瓦/Valorant。P 社和大型模拟策略游戏（群星、维多利亚、欧陆风云等）也按 CPU 缓存/后期模拟帧优先处理。

- 预算允许时优先高采用率 X3D 候选：AM4 以 5800X3D 为主，AM5 以 7800X3D、7950X3D、9800X3D、9850X3D、9950X3D、9950X3D2 为主；能上 9800X3D 时不默认退到 7800X3D，更高预算且兼顾生产力时再考虑 Ryzen 9 X3D。其他 X3D 型号只有在价格优势明确或用户点名时才作为备选，不因名称带 X3D 就自动提高权重。
- 显卡按分辨率和刷新率“够用并留余量”，不要为了旗舰显卡牺牲 CPU、内存延迟和散热稳定性。2K FPS/吃鸡/CS2 这类场景可用 RTX5070/5070Ti 配平时，不强行堆到 RTX5080。
- 内存优先低延迟双通道：AM5 默认 DDR5-6000 CL30/32，Intel 可看 DDR5 6400-7200 或成熟 DDR4 3600/3200 取性价比。
- 输出要关注 1% low、CPU 限制、温度稳定性和显示器刷新率，不只看平均帧。

## 本地 AI 编程/终端 agent 硬件配置

用户明确是在为本地 AI 编程/终端 agent 工作流配电脑、升级电脑或评估硬件需求时，才进入本节。典型说法包括“给 Codex/Claude Code/opencode/OpenClaw/Hermes 配电脑”“多 agent 并发主机”“本地项目索引/离线知识库/大量工具链并发吃配置吗”。如果用户只是问如何安装、运行或使用某个 agent、插件、命令或工作流，不触发装机配置。

- 优先级为内存、SSD、CPU，其次才是显卡；但只要用户场景和 AI 辅助、embedding、向量库、本地小模型、ComfyUI 或视频生成有关，预算允许时默认保留一张 NVIDIA 显卡作为 CUDA/本地模型弹性，不把 Intel Arc 作为 AI 相关默认显卡。用户明确纯云端、纯编译或极限省钱时，才降低显卡权重。
- 官方最低运行门槛不等于装机推荐。Claude Code 官方最低可到 4GB+ RAM；OpenCode/Codex 这类终端 agent 主要依赖网络模型和本地项目工具链。实际装机按“IDE + 浏览器 + 终端 agent + 索引/缓存 + Docker/测试”的并发负载估算。
- 默认 32GB 起步；多项目、多容器、浏览器和 IDE 常驻建议 64GB。多 agent 并发、本地向量库/知识库、多个沙箱或后台服务常驻，可建议 96GB/128GB；明确容量时用 `--min-capacity N --max-capacity N --sort tier`（N 为 64、96 或 128）收窄，再看频率、时序、套条数量和候选池权重。不要为纯 agent 盲目牺牲 SSD 和 CPU。
- SSD 优先 2TB 主流 TLC NVMe；大量索引、缓存、日志、向量库、模型或多仓库时建议第二块 SSD 或 4TB。明确容量时同时用 `--min-capacity` 和 `--max-capacity`，再用 `--sort tier` 比较同容量候选；优先看 TLC、耐久和持续写入，不要用 QLC 系统盘。
- CPU 优先高单核和足够核心数，Intel 带核显对排障/编码有价值；AMD 也可用，按预算和平台扩展性取舍。
- 如果用户实际想跑本地 LLM、embedding 或重排序模型，再单独进入本地模型路线，显存/内存需求会明显上升。

## ComfyUI / Stable Diffusion / 文生视频

用户说 ComfyUI、SD、Stable Diffusion、文生图、LoRA、本地绘图、文生视频、AnimateDiff、Flux、SVD 等，必须按本地生成式 AI 路线处理。

- 必须优先 NVIDIA 显卡和显存；同预算先保显存和 CUDA 生态，再看 CPU 核心数。AMD/Intel 显卡只有用户明确要求或预算极低时作为取舍，并说明生态风险。
- 文生图入门至少 12GB 显存更稳；文生视频、较大模型、多 ControlNet/高分辨率优先 16GB/24GB+ 显存。
- RTX5070Ti 和 RTX5080 都是 16GB 显存时，不要只因型号更高自动上 RTX5080；需要比较价格、显存带宽、功耗、散热、SSD/内存/主板是否被挤压。文生视频或重度生成仍优先显存容量和带宽，预算不够时可把 RTX5070Ti + 更大 SSD/更好电源作为均衡方案。
- 内存默认 32GB 起步，文生视频、多模型常驻或生产力混用优先 64GB；明确要 64GB 时用 `--min-capacity 64 --max-capacity 64 --sort tier`，避免被 32GB 低价候选或更大容量候选带偏。
- SSD 默认 2TB 主流 TLC NVMe；模型库、素材、缓存较多时优先 4TB 或双 SSD。明确容量时同时用 `--min-capacity` 和 `--max-capacity`，再用 `--sort tier` 比较同容量候选，且避免 QLC 系统盘。
- CPU 够用即可，但不要用太低端 CPU 拖慢预处理、解码、批处理和多任务。15000 元以上且显卡/显存没有把预算卡死时，至少比较 R7 9700X 或同档 CPU；如果 9700X 只小幅超预算或能通过非核心外观件回收预算，优先给 9700X。R5 9600X 只作为保显卡/保显存/保 64GB 内存或大 SSD 的压预算取舍，并在报告里写出 9700X 替代价差。

## CAD / 建模绘图

用户只说 CAD 时，先问或自行列出假设：AutoCAD 2D、AutoCAD 3D、SolidWorks/Inventor/Revit、装配规模、是否专业商用和是否需要认证显卡。

- AutoCAD 2D/轻 3D 优先高频 CPU、32GB 内存和主流 TLC SSD；显卡不用盲目堆高，基础独显或中端 NVIDIA 足够。
- AutoCAD 大型 3D、点云、多屏和复杂模型提高显卡和内存优先级，但仍保高单核 CPU。
- SolidWorks/复杂装配优先高频 CPU、32GB/64GB 内存、SSD；装配越大内存越优先。
- 专业商用、需要官方支持、RealView/Ambient Occlusion 或复杂装配视口时，说明 RTX PRO/工作站卡的价值；预算有限或兼顾游戏时可用 GeForce，但要明示认证和支持取舍。
- CAD 工作站不建议超频式不稳定方案；电源、散热、主板供电要保守。

## PS / Lightroom / 图片后期

用户说 PS、Photoshop、Lightroom、修图、摄影后期、批量出片时，按图片后期硬件路线处理。

- 高频 CPU 优先，核心数不需要无限堆；预算足够优先较新的 Ryzen 7/9、Intel i5/i7/Core Ultra 中高频型号。
- 内存 32GB 起步；大尺寸 RAW、全景、堆栈、多图层 PSD 建议 64GB。
- SSD 优先主流 TLC NVMe，系统/软件和素材可一块 2TB 起步；素材量大时加第二块 SSD 或大容量仓库盘。
- 显卡需要支持并稳定，但纯修图不为旗舰显卡牺牲 CPU/内存/SSD；中端 NVIDIA/AMD 独显通常足够，多 4K 显示器或 AI 滤镜再提高显存。

## PR / 达芬奇 / 视频剪辑调色

用户说 PR、Premiere、视频剪辑、B站剪辑、4K 剪辑、达芬奇、DaVinci、调色时，先问或假设素材分辨率、编码、位深、特效、代理和是否多软件并开。

- Premiere/H.264/H.265 素材优先 Intel 带核显 + NVIDIA 显卡，兼顾 Quick Sync 和 NVENC；RAW/多特效/多软件时提高 CPU、内存和显卡档位。
- Core Ultra 200S 平台优先 Core Ultra 5 250K/250KF Plus、Core Ultra 7 270K 或库内同档 Core Ultra 7；Core Ultra 9 只在重度多任务、复杂工程或预算明确足够时使用。若 U9 285K 导致超预算或挤压内存、SSD、散热和机箱风道，优先比较 U7 270K Plus/同档 U7。
- DaVinci Resolve 更偏 GPU/显存，调色、降噪、Fusion、AI 功能优先 NVIDIA 和显存；4K 建议 12GB+ 显存，6K/8K 或重降噪建议 16GB/20GB+。
- 内存按分辨率给足：1080p 可 32GB 起步，4K 剪辑建议 64GB，6K/8K 或 AE/PS/PR/达芬奇并开建议 96GB/128GB+。
- SSD 不只看容量：至少 2TB 主流 TLC NVMe；预算足够时分系统/软件、素材项目、缓存/暂存盘。高码率素材和缓存优先高速 TLC，不用 QLC 系统盘。
- 主板也是剪辑平台重点：素材导出、外置高速盘、采集卡、多硬盘和缓存盘会吃 I/O。AMD 优先 X870/X870E 并复核 USB4、M.2、SATA；Intel 优先 Z790/Z890 或同等级高扩展主板，并复核高速 USB/雷电扩展、M.2、SATA 和 PCIe 插槽。
- 电源、散热和机箱风道按长时间渲染负载留余量，不用刚好够的电源和入门散热压高端 CPU/GPU。

## AE / 动效合成

用户说 AE、After Effects、动效、合成、MG 动画时，按 CPU、内存和缓存盘优先处理。

- CPU 对整体体验很关键，优先高性能新平台；大量渲染和多软件并开再提高核心数。
- 内存 64GB 作为主流建议；4K 以上、复杂合成、多软件并开或长时间预览建议 128GB。
- 高速 TLC NVMe 缓存盘很重要；预算允许时把缓存/项目盘与系统盘分开。
- 2D 动效不盲目堆显卡；3D、Advanced 3D、粒子、GPU 插件较多时优先 NVIDIA 和更大显存。

## Blender / 3D / 渲染

用户说 Blender、C4D、Maya、3D 建模、动画、渲染时，先按建模动画、GPU 渲染、CPU 渲染、流体/布料仿真分流。

- 建模和动画看高频 CPU、稳定视口和足量内存；不需要为建模本身堆极高核心数。
- GPU 渲染优先 NVIDIA、CUDA/OptiX 和显存；项目复杂度不明时宁可提高显存，不要只看核心性能。
- CPU 渲染、流体仿真、烘焙等吃多核心时，提高核心数和散热，预算高再考虑 HEDT/工作站平台。
- 内存小项目 32GB 起步，大场景/贴图/仿真/多软件建议 64GB/128GB。
- SSD 至少 2TB 主流 TLC NVMe；素材库、贴图库、缓存和工程多时加第二块 SSD 或大容量仓库盘。

## UE / Unity / 游戏开发

用户说 UE、Unreal、Unity、游戏开发、虚幻、Shader 编译、打包时，按引擎开发工作站处理。

- CPU 是编译代码、Shader、烘焙和导入资源的关键；项目越大越需要更多核心，但普通用户不默认上 HEDT。
- AMD 非 X3D 多核心路线预算足够时优先 Ryzen 9 9950X；如果预算不足或更看重单 CCD 稳定表现，可用 Ryzen 7 9700X。Ryzen 9 9900X 只有在价格优势或核心数收益明确时作为取舍。
- GPU 决定编辑器视口、实时预览、光追和目标画质；UE5/大型场景优先 NVIDIA 和 12GB/16GB+ 显存。
- 内存 32GB 为入门，较大项目、烘焙、多工具并开建议 64GB+。
- SSD 默认 2TB 主流 TLC NVMe；大型工程、Derived Data Cache、素材库和构建输出多时建议第二块 2TB/4TB SSD。
- 若用户同时玩 3A，先满足开发项目规模，再按游戏分辨率补显卡档位。

## 高预算配平

高预算仍要均衡，不把钱浪费在装饰件或重复堆料上。

| 预算 | 默认方向 | 注意 |
|------|----------|------|
| 15000 | 游戏/FPS 优先 9800X3D + RTX 5070 Ti/5080 档；32GB DDR5-6000 低延迟；2TB 主流 TLC SSD；850W+ ATX 3.x 金牌电源 | 2K FPS 不强行上 5080；如果 5080 超预算，优先保 9800X3D、显卡档位和电源下限，外观件降级 |
| 20000 | 9800X3D + RTX 5080 高规格；2TB/4TB 高端 TLC SSD；32GB/48GB 高接受度内存；850W/1000W ATX 3.x 电源按静音和升级余量取舍 | 纯游戏不默认上 9950X3D；RTX5080 通常 850W 足够，1000W 属富余或升级余量；生产力/直播剪辑再考虑更多核心和 64GB 内存 |
| 25000+ | 4K 高刷/生产力按 RTX 5090/5090D、9950X3D/高端 Intel 分流 | 必须核对机箱风道、电源接口、显卡长度和散热，不用低端板电源硬拖旗舰件 |

预算分配原则：CPU 和显卡优先，CPU 够用但不能拖显卡；主板、电源、散热、机箱是稳定性底线；SSD/内存按容量和可靠性足够即可，不为了跑分挤占显卡预算。纯游戏 32GB DDR5 超过 1500 元必须解释价值；没有用户指定、48GB/64GB 或特挑低时序理由时，必须改选 800-1300 元主流套条，把钱留给显卡、电源和散热。

用户明确 ROG 全家桶、华硕全家桶、白色旗舰或展示向整机时，可以把高价 ROG/TUF/同生态配件作为主方案，但要把溢价说明成人能理解的取舍：外观统一、生态联动、散热规格、屏幕/灯效或品牌系列，而不是性能必需。高价不设硬上限，低价异常仍要复核。预算明显不足以同时保证核心性能、电源、散热和机箱风道时，不硬凑零散旗舰件；先说明预算缺口，再给同生态非旗舰、TUF/PRIME 或性能优先替代。
