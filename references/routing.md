# 需求路由

先识别用途，再分配预算。用户有明确偏好时优先；否则按默认路由。

## 口语化需求理解

用户常用装机圈表达 → 路由参数映射：

| 用户说 | 含义 | 路由参数 |
|--------|------|---------|
| 黑色海景房 | 黑色 + 展示型机箱(玻璃侧透) + 可能水冷 | color=black, case_type=showcase |
| 白色海景房 | 白色 + 展示型机箱 + 白色配件 + 可能RGB | color=white, case_type=showcase |
| 无光 / 不要灯 | 不花预算在RGB/灯效上 | rgb=false |
| 纯性能不要颜值 | 性能优先，压外观预算 | aesthetics=false |
| 水冷 / 360水冷 | 用水冷散热 | cooler_type=liquid, radiator=360 |
| 小机箱 / ITX | 紧凑型，MATX/ITX | form_factor=MATX/ITX |
| 低U高显 | CPU够用，显卡拉满 | routing=low_cpu_high_gpu |
| 2K / 4K | 分辨率目标，影响显卡档位 | resolution=2K/4K |
| 多开 / 挂机 | CPU核心+内存容量优先 | routing=multitask |

## 默认路由

- 3A: 低U高显，优先显卡/显存/散热；CPU够用即可。
- 网游: CPU和显卡均衡，优先高频低延迟内存。
- 办公: 高U低显或核显，看CPU/多屏/内存/SSD/静音。
- 生产力: 按软件分流（剪辑/AI/渲染是否吃GPU）。
- 多开: CPU核心/内存容量/SSD/电源稳定性优先。

## 选件优先级

先满足用途和预算，再看外观。不要把每个品类的最低价候选直接拼成整机。

1. 优先选择数据完整的配件: 有 `price_cny`/`price_status`/`price_date`，并且关键兼容字段不为空。
2. 避开硬件范围外的老平台和老规格: DDR3、120/128GB SATA SSD、Intel 11代及更早、AM4、RTX 40/RX 7000 等不作为新装机首选。
3. 最终整机报价应尽量贴近用户预算: 常规目标为预算的 92%-102%；低预算可到 90%，但必须说明取舍。
4. 如果为了完整计价、稳定性或兼容性必须超预算，明确给出"可用方案"和"压预算方案"的差异，不要假装刚好卡进预算。
5. 查询候选时按品类分开跑，通常用 `--limit 50` 起步；高预算或白色/海景房等强约束场景用更大的 limit 后二次筛选。
6. 中高端装机查询显卡和主板时用 `--sort tier`: 显卡按芯片等级优先（如 RTX 5070 优先于 RTX 5060Ti），主板按芯片组等级优先（B760/B650 优先于 H610/A620），同等级内按价格升序。
7. 候选收窄按依赖顺序做，不要每类各取最低价后拼单:
   - 选主板时用已选 CPU 的 socket/供电需求过滤；选 CPU 时反向核对已选主板 socket。
   - 选内存时用主板 DDR 代际、可支持频率、槽位和容量上限过滤。
   - 选电源时用已选整机功耗加余量过滤，不只看显卡推荐瓦数。
   - 选机箱时用主板版型、显卡限长、散热限高/冷排位和电源规格过滤。
   - 最后再用 `check_compatibility.py --strict` 做全单校验。

## 预算配平参考

这些是软约束，用来避免低价候选把整机拉偏，不是固定模板。

| 预算 | 配平重点 |
|------|---------|
| 4000 | 显卡约 35%-45%；CPU+主板约 25%-30%；必须保留机箱/电源/散热预算。稳定优先时 NVIDIA 路线可超预算，需说明。 |
| 5000 | 优先 6 核 CPU + 12GB/8GB 主流显卡 + 1TB SSD；不要用小容量 SATA SSD 凑价。 |
| 7000 | 2K 3A 优先显卡和 32GB 内存；整机小计不要在未含机箱时就贴满 7000。 |
| 8000 | 白色/海景房有外观溢价，应把显卡压在 5060Ti/同档而不是退到入门卡。 |
| 10000 | RTX 5070/同档显卡优先，电源必须留 16pin/12V-2x6 余量。 |
| 12000 | 游戏+生产力优先 8 核 CPU、32GB 起步、2TB SSD；重度剪辑提示 64GB 可能超预算。 |
| 16000+ | 游戏优先可选 X3D + RTX 5080/高规格显卡；不要把预算花在装饰件上。 |

4000 元 3A 属于入门新件边界，目标是贴近 4000 元和 1080p 可玩，不要强称 2K/高画质。为贴预算可接受 16GB 内存、Arc B570/RTX5050/同档显卡，并明确后续升级 32GB 的取舍。

8000 元白色海景房优先保显卡档位和整机预算贴近；如果为了白色件/海景房外观回退到 DDR4 或较低 CPU，需要把它写成取舍，不要说成同价位最优平台。

## 品牌优先级

选件时按品牌梯队优先，低预算或特殊需求可降级。同梯队内50元差价保持随机性，避免每次都选同一品牌。

### 主板

1. **芯片组门槛**: 除非预算极低（主机¥4000以下），不选入门芯片组。
   - **不推荐**: Intel H610/H810、AMD A620/A820 — 供电弱(6相)、无VRM散热装甲、M.2口少、带不动i5以上CPU。
   - **优先推荐**: Intel B760/B860、AMD B650/B850 — 供电充足(8+1相起)、有散热装甲、M.2口2+、内存超频支持。
   - 社区验证: "H610搭配i7会因供电不足降频"(百度百科)、"我不推荐H610主板"(B站5.2万播放)。
   - 查询提示: 5000 元及以上整机不要按主板低价升序首屏直接选件；用 `query_components.py --category mb ... --sort tier`，再按 CPU socket、DDR 代际和价格二次筛。
2. **一线（御三家）**: 华硕 > 技嘉 > 微星 — 预算足够时必选。
   - 华硕: 高端市场占有率超40%，BIOS最成熟，QVL最全。推荐系列: TUF GAMING(重炮手)、PRIME。
   - 技嘉: 性价比最好，AORUS系列稳定。推荐系列: GAMING X(魔鹰)、AORUS ELITE(小雕)。
   - 微星: 供电设计强，内存兼容性好。推荐系列: **MORTAR(迫击炮)**、MAG、PRO。
3. **二线**: 华擎 — 仅预算紧或特定需求时选。推荐系列: Steel Legend(钢铁传奇)、PG-ITX/TB4。
4. **三线/入门**: 七彩虹(iGame系列)、铭瑄、精粤、昂达 — 低预算(¥500以下主板)或白色需求精粤无替代时可选。
   - **注意**: 精粤等小品牌BIOS更新慢、QVL覆盖少，高预算(¥8000+)不应首选。
5. **微星型号避雷**:
   - **避雷**: 微星 BOMBER(爆破弹) 供电弱(80W)、无VRM散热片，带不动i5以上CPU。
   - **推荐**: 装机社区认可的是 **MORTAR(迫击炮)** 系列（不是 BOMBER(爆破弹)），两者中文易混淆但完全不同。

### 显卡

1. **芯片优先**: NVIDIA优先（市场份额94%，驱动稳定性好），除非用户明确要求AMD。
2. **NVIDIA品牌梯队**:
   - 一线: 华硕 > 技嘉 > 微星 > 七彩虹 — 七彩虹为中国出货量第一，性价比高。
   - 二线: 影驰、铭瑄、盈通、万丽、电竞叛客、映众 — 预算不够一线时选。
   - 参考: 英伟达原版(Founders Edition) — 供电/散热保守，不优先推荐。
3. **AMD品牌梯队**: 蓝宝石 > 撼讯(PowerColor) > 其他。
4. **Intel品牌梯队**: 蓝戟(Gunnir) — Arc显卡首选品牌。

### 硬盘 (SSD)

1. **只推荐 PCIe 接口固态**: 除非用户明确要求，不推荐机械硬盘(HDD)。
2. **优先 TLC 颗粒**: TLC寿命3000次P/E，QLC仅1000次，缓外速度差距大。不推荐QLC。
3. **默认一块硬盘**: 不拆成多块小容量。默认容量 1TB。
4. **容量规则**: 1万元以下预算默认1TB；1万元以上可出2TB。
5. **品牌**: 三星、西数、铠侠、致态、海力士优先；金士顿、雷克沙、英睿达次选。

### 电源

1. **高端显卡供电字段优先**: RTX 5070/RTX 5070 Ti 及以上，优先选 `native_16pin_gpu_power=true` 或脚本严格检查能通过的电源。
2. **不要只看型号文字**: 有些型号名写 ATX3.1、PCIe5 或原生线，但离线库字段缺失时严格检查会警告；这类只能作为备选并提示复核。
3. **预算内留余量**: 电源功率以 `check_compatibility.py --strict` 的推荐功率和余量为准，不只按显卡推荐瓦数估。

### 新平台优先

- 预算足够时优选新芯片组: B850优先于B650，X870优先于X670，Z890优先于Z790。
- 新平台优势: 原生支持新CPU(免刷BIOS)、PCIe 5.0满血、USB4/WiFi7、未来扩展性。
- B650/X670需刷BIOS才能用Ryzen 9000，对新手多一步。
- 但B650降价后性价比高，预算紧张时可选，需标注"需刷BIOS"。

### 品牌随机性

- 同梯队、50元差价内的配件，不要每次都选同一品牌。
- 避免输出千篇一律的配置方案，保持品牌多样性。

## 色系和外观

- 黑色: 优先黑色/默认黑色。
- 白色: 必须白色型号或明确白色版本。
- 海景房: 展示型机箱(玻璃侧透)，需要查冷排位、显卡限长、白色件价格溢价，并预留风扇预算。
- 不要RGB: 不花预算在灯效/海景房/白色溢价和装饰风扇上。

## 机箱和风扇

- 机箱是必选配件，必须计入总价。优先选有价格的兼容机箱；无价格机箱只能作为备选并标注总价不完整。
- 海景房默认需要补风扇。若用户没有指定水冷，按机箱风扇位预留 3-6 把风扇预算；若用了水冷，冷排自带风扇可抵扣对应风扇位。
- 离线库风扇数据不足时，不要编具体风扇型号；用"风扇预算预留"写入总价或备注，并建议下单前确认机箱自带风扇数量。

## 平台

- AM5 只配 DDR5。
- Intel 12/13/14代区分 D4/D5 主板。
- 不混搭 DDR4 和 DDR5。

## 内存选择规则

内存选择按平台分流，优先保容量和频率，其次看颗粒和时序。

### 通道规则（通用）

1. **优先双通道（2条套）**: 默认推荐 2 条套装（如 16GB×2=32GB），不推荐 4 条。
2. **4条需明确要求**: 用户明确要求 4 条或大容量（如 64GB=16GB×4）时才用 4 条。
   - DDR5 插 4 条和 2 条都跑双通道，4 条不会变四通道，但可能增加主板内存控制器压力、降低极限超频稳定性。
3. **预算不足时可单条**: DDR5 单条本身就是双通道（2×32-bit），单条性能衰减比 DDR4 时代小。
   - 但双条（满血 4×32-bit）带宽仍优于单条，游戏最低帧更稳。
   - AMD X3D 处理器因大缓存，单条性能衰减几乎可忽略，预算不足时单条可接受。
   - 非 X3D 处理器预算不足时，宁可降容量也要保双条（如 2×8GB > 1×16GB）。

### AMD AM5 平台 (DDR5)

1. **甜点频率优先**: 6000MHz 为甜点位（Infinity Fabric 1:1同步极限），时序优先 C28 > C30 > C32。
2. **颗粒优先级**: 海力士新M-Die(24G/48G大容量) > 海力士A-Die > 海力士旧M-Die > 三星 > 镁光 > 国产。
   - 新M-Die: 24G×2=48G 大容量+超频潜力好，性价比高
   - A-Die: 超频上限高，C28低时序首选
   - 旧M-Die: 6000-6400主流，部分新批次C28难压
   - 镁光: 已退出消费端市场(2026.02)，货源减少
3. **预算不足时保序**: 16G容量 > 6000频率 > 冗余容量 > 时序。
   - 先保 16GB（2×8GB）容量能用
   - 再保 6000MHz 频率
   - 再考虑升到 32GB（2×16GB）
   - 最后优化时序（C28 > C30 > C32）
4. **容量上限**: 默认 32GB（2×16GB）为止。A-Die 32G 或新M-Die 48G 后不再增加，除非：
   - 本地 AI / 大模型推理
   - 多开 / 挂机 / 虚拟机
   - 预算充足且用户明确要求
5. **频率上限注意**: 超过6000MHz对AMD收益极小（异步分频），不建议为超频多花钱。

### Intel 平台 (DDR4/DDR5)

1. **频率越高越好**: Intel 对内存频率敏感，高频率收益明显。
   - DDR5: 14代Core推荐 6400-7200；Core Ultra 200S推荐 7200-8000+（需Z890主板支持）
   - DDR4: 3600 > 3200 > 2666
2. **颗粒优先级**: 同 AMD — 海力士新M-Die > A-Die > 旧M-Die > 三星 > 镁光 > 国产。
3. **预算不足时保序**: 16G容量 > 高频率 > 时序 > 冗余容量。
   - 先保 16GB 容量
   - 再拉高频率
   - 再优化时序
   - 最后考虑冗余容量
4. **容量上限**: 同 AMD — 默认 32GB 为止，A-Die 32G 或新M-Die 48G 后不再增加，除非特殊场景。

### 特殊场景例外

- **本地 AI**: 64GB+ 有意义（模型加载需要大内存），可超 32GB 上限
- **多开/挂机**: 64GB+ 有意义（每个实例占内存），可超上限
- **生产力/剪辑**: 32-64GB 按软件需求决定
- **纯游戏**: 32GB 足够，不需要更多
